## 推荐志愿操作接口文档

### 1. 加入志愿接口
- **接口地址**：`http://localhost:8080/application/save`
- **前端请求地址**：`/api/application/save`
- **请求方式**：`POST`
- **说明**：在智能志愿推荐页点击“加入志愿”时调用，将推荐院校/专业写入志愿填报列表。

#### 请求参数（Body，JSON）
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| studentId | String | 是 | 学生编号 |
| universityId | String | 是 | 院校ID/代码 |
| universityName | String | 是 | 院校名称（便于后端落库） |
| majorId | String | 是 | 专业ID/代码 |
| majorName | String | 是 | 专业名称 |
| recommendationId | String | 否 | 推荐记录ID（若有） |
| volunteerOrder | Number | 否 | 默认志愿顺序，前端当前写入 1，可在志愿管理中再调整 |

#### 成功响应示例
```json
{
  "code": 200,
  "message": "保存成功",
  "data": null
}
```

#### 失败响应示例
```json
{
  "code": 500,
  "message": "学生信息不存在",
  "data": null
}
```

### 2. 推荐详情接口
- **接口地址**：`http://localhost:8080/recommendation/volunteer/detail`
- **前端请求地址**：`/api/recommendation/volunteer/detail`
- **请求方式**：`GET`
- **说明**：点击“查看详情”弹窗展示数据。

#### 请求参数（Query）
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| studentId | String | 是 | 学生编号 |
| universityId | String | 是 | 院校ID/代码 |
| majorId | String | 是 | 专业ID/代码 |

#### 成功响应示例
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "universityId": "10001",
    "universityName": "北京大学",
    "majorId": "080901",
    "majorName": "计算机科学与技术",
    "lastYearScore": 680,
    "scoreDifference": -29.5,
    "admissionProbability": 42,
    "recommendationRate": 4.5,
    "advantageAnalysis": "该校该专业在算法研究方向领先，科研资源丰富。",
    "riskAdvice": "分数差距较大，建议作为冲刺志愿。",
    "suggestion": "若想提高录取概率，可同步填报同层次计算机类专业。"
  }
}
```

#### 失败响应示例
```json
{
  "code": 404,
  "message": "未找到对应推荐数据",
  "data": null
}
```

#### 说明
- 详情接口返回的数据会映射到弹窗中的“优势分析/风险提示/录取建议”等区域。
- 若 `admissionProbability` 或 `recommendationRate` 字段不存在，前端会显示默认占位内容。
